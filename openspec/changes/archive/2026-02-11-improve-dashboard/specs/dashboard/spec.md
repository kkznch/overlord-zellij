## ADDED Requirements

### Requirement: 全ロールのステータスをテーブル表示
ダッシュボードは6つの全ロールのアイコン・名前・ステータス・タスク・最終更新からの経過時間をテーブルで表示しなければならない (SHALL)。

#### Scenario: ステータステーブルの描画
- **WHEN** ダッシュボードが起動しリレーストアにステータスデータがある時
- **THEN** 6行のテーブルが表示される（overlord, strategist, inferno, glacier, shadow, storm）
- **THEN** 各行に絵文字アイコン・ロール名・ステータスシンボル+テキスト・タスク説明・経過時間が表示される

#### Scenario: ステータスの色分け
- **WHEN** ロールのステータスが "working" の時
- **THEN** ステータスセルが緑色で表示される
- **WHEN** ロールのステータスが "blocked" の時
- **THEN** ステータスセルが赤色で表示される
- **WHEN** ロールのステータスが "done" の時
- **THEN** ステータスセルがシアン色で表示される
- **WHEN** ロールのステータスが "idle" の時
- **THEN** ステータスセルが暗灰色で表示される

### Requirement: 未読メッセージインジケータ
未読ペンディングメッセージがあるロール名の横に `*` マークを表示しなければならない (SHALL)。

#### Scenario: ペンディングフラグがあるロール
- **WHEN** リレーストアにロールのペンディングフラグファイルが存在する時
- **THEN** テーブル内のロール名の末尾に ` *` が表示される

#### Scenario: ペンディングフラグがないロール
- **WHEN** ロールにペンディングフラグがない時
- **THEN** ロール名にマーカーは表示されない

### Requirement: ワーカー数サマリー
稼働中のロール数とペンディングメッセージのあるロール一覧をサマリー行として表示しなければならない (SHALL)。

#### Scenario: サマリー表示
- **WHEN** 6ロール中3つが "working" で2つにペンディングメッセージがある時
- **THEN** サマリー行に "Workers: 3/6  Pending: <role1> <role2>" と表示される

### Requirement: 最新メッセージフィード
全受信箱から最新5件のメッセージをタイムスタンプ付きで表示しなければならない (SHALL)。既読マークは付けない。

#### Scenario: タイムスタンプ付き最新メッセージ
- **WHEN** リレーストアにメッセージが存在する時
- **THEN** 最新5件が新しい順に表示される
- **THEN** 各メッセージにローカルタイムスタンプ (HH:MM:SS)・送信元・送信先・件名が表示される

#### Scenario: メッセージなし
- **WHEN** リレーストアにメッセージが存在しない時
- **THEN** 最新メッセージセクションは空（プレースホルダーテキストなし）

### Requirement: 自動リフレッシュポーリング
2秒ごとにリレーストアをポーリングし画面を再描画しなければならない (SHALL)。

#### Scenario: 定期リフレッシュ
- **WHEN** 前回の描画から2秒が経過した時
- **THEN** リレーストアから最新データを取得して画面を再描画する

### Requirement: q キーで終了
ユーザーが `q` キーを押した時、ターミナルを元の状態に復元してクリーンに終了しなければならない (SHALL)。

#### Scenario: クリーン終了
- **WHEN** ユーザーが `q` を押した時
- **THEN** raw mode が無効化され、代替画面から離脱し、プロセスが終了コード 0 で終了する
